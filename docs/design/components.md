# MythOS Components

Version: 0.1


# 1. Introduction

Components are reusable building blocks of MythOS interfaces.

Every application should use these components.


# 2. MythButton


Purpose:

Primary user interaction.


Characteristics:

- Rounded corners
- Smooth hover
- Cyan active state


States:

- Default
- Hover
- Active
- Disabled


# 3. MythCard


Purpose:

Information container.


Characteristics:

- Floating surface
- Glass effect
- Soft shadow


Used for:

- Dashboard
- Settings
- Information panels


# 4. MythWindow


Purpose:

Application container.


Characteristics:

- Rounded corners
- Transparent surfaces
- Modern controls


# 5. MythDialog


Purpose:

User decisions.


Examples:

- Confirmations
- Warnings
- Actions


# 6. MythNotification


Purpose:

System communication.


Types:

- Information
- Success
- Warning
- Error
- AI


# 7. MythInput


Purpose:

Data entry.


Characteristics:

- Minimal design
- Clear focus state
- Cyan highlight


# 8. Component Rule

No application should create its own visual language.

All interfaces must use Myth components.
